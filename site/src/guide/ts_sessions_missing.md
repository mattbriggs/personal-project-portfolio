# Sessions Not Appearing on the Dashboard or Sessions Tab

Sessions you have created do not appear in the Sessions tab or are not reflected in the Dashboard score.

One or more of the following occurs:

-   The Sessions tab shows no sessions for the current week even though sessions were created.
-   The Dashboard shows 0 planned or done sessions for a project that has sessions.
-   Marking a session Done does not change the project score.

**The week selector is showing a different week than the one the sessions were created for.**

1.  In the Sessions tab, check the week key displayed in the week selector.

2.  Compare it with the week key shown in the left panel for the current week.

3.  If they differ, use the arrow buttons in the week selector to navigate to the correct week.


**The session's scheduled date falls in a different week than expected.**

1.  Locate the session: in the Sessions tab, click the Previous button to navigate to surrounding weeks, or look for the session in the week where you created it.

2.  Select the session and click **Edit Session**.

3.  Correct the **Date** field to a date in the intended week and save.


**The session status is Backlog or Cancelled.** Backlog and Cancelled sessions do not appear in the Dashboard counts and do not contribute to the project score.

1.  Select the session in the Sessions tab.

2.  Click **Edit Session**.

3.  Change the status to **Planned** and save.


**The project's status is Backlog or Archive.** Sessions for non-active projects are excluded from the Dashboard.

1.  Click the **Projects** tab.

2.  Set the status filter to **All** and locate the project.

3.  Click **Edit Project**, change the status to **Active**, and save.
