# Create a Milestone

Add outcome-based milestones to a project to define what done looks like at each meaningful stage of the work.

Milestones represent completed outcomes, not tasks. Describe each milestone as a state that will be true when a phase of work is finished: "First draft complete," not "Write first draft."

1.  Click the **Milestones** tab.

2.  In the project dropdown at the top of the tab, select the project you want to add a milestone to.

3.  Click **New Milestone**.

    The Milestone dialog opens.

4.  In the **Description** field, enter the outcome name.

    Write this as a noun phrase or past-tense statement that describes what will be true when this phase is complete. Examples:

    -   "Outline approved"
    -   "All episodes recorded"
    -   "Prototype tested with five users"
5.  Set the **Status** field.

    -   Select **Backlog** for a future milestone not yet in scope.
    -   Select **Planned** for a milestone you are actively working toward.
6.  \(Optional\) Enter a **Target Date** in YYYY-MM-DD format.

7.  \(Optional\) Enter any notes that help clarify what this milestone means or how you will know it is achieved.

8.  Click **Save**.

    The milestone appears in the Milestones list for the selected project.

## Reorder milestones by name

Click the **Milestone** column heading to sort the list alphabetically. Clicking again reverses the sort order \(▼\). A third click returns to the original order. You can rename a milestone to use a numbered prefix \(for example, "1. Outline approved", "2. First draft complete"\) so that clicking the heading sorts them in your intended sequence.
